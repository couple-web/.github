## 🖼️ 배경

현대 커플들은 서로의 일상을 기록하고 공유하는 다양한 플랫폼을 사용하고 있지만,  
실제 연인 관계에 최적화된 기록과 추천, 소통 기능은 부족한 실정입니다.  

기존 커플 앱들은 단순한 다이어리 기능에 그치는 경우가 많아  
더 감성적이고, 더 실시간적인 기능을 원하던 사용자의 니즈를 발견했습니다.

이에 저희는 **연인만을 위한 감성적 공간**을 제공하고자  
기록, 추천, 소통을 한데 모은 "AI 기반 커플 플랫폼" WE ARE를 기획하게 되었습니다.

---

## 📊 개요

WE ARE는 단순한 커플 다이어리나 메신저를 넘어서,  
**연인의 기억을 감성적으로 기록하고**,  
**AI 기반으로 데이트 장소를 추천하며**,  
**기념일 및 일정 알림을 실시간으로 받을 수 있는**  
커플을 위한 통합 플랫폼입니다.

또한, 기존 커플 앱의 단조로움을 벗어나  
**사용자의 라이프스타일과 감성에 맞춘 맞춤형 추천과 실시간 알림 서비스**를 통해  
더욱 자연스럽고 실용적인 관계 중심 플랫폼을 지향합니다.

# 🧩 핵심 기능

### 📸 커플 기록 앨범
- 연인과의 사진, 메모, 동영상을 감성적으로 기록할 수 있는 앨범형 UI 제공
- 날짜 기반 자동 정렬, 추억 회고에 최적화된 타임라인 구성
- 앨범 콘텐츠에 '좋아요', 댓글, 대댓글 기능 제공
- 일정에 등록된 날짜의 날씨 정보를 자동으로 표시하여 추억을 감각적으로 기록

### 🗓️ 커플 캘린더 & 기념일 관리
- 기념일, 투두 일정 등록 및 수정 가능
- D-Day, 반복 일정 기능 포함
- 월간/주간 캘린더 뷰 제공

### 📍 데이트 장소 추천 시스템
- 카테고리(카페, 전시, 맛집 등) + 지역 필터로 커스터마이징된 장소 탐색 가능
- Google Map 연동으로 지도 기반 UI 제공
- 추천 장소 클릭 시, 상세 정보(사진, 설명, 지도) 카드로 표시

### 💬 실시간 소통 & 알림
- 피드, 댓글, 기념일 등 주요 사용자 이벤트에 대해 실시간 알림 제공
- 알림은 개별 사용자에게 비동기 처리되어 개인화된 UX 제공
- 알림 목록에서 읽음 여부 확인 및 알림 삭제 가능
- 일정 3일 전 기준으로 OpenWeather API 연동, 지역 기반 날씨 알림 자동 발송

### 🙋‍♂️ 사용자 프로필
- 닉네임, 프로필 사진, D-Day 수정 기능
- 팔로워 ,팔로잉 신청을 통해 관계 형성
- 마이페이지에서 내가 작성한 콘텐츠, 댓글, 일정, 즐겨찾기 모아보기 제공

### 🧑‍💻 관리자 기능
- 관리자 전용 대시보드 제공 (통계 , 콘텐츠 관리)
- 차트.js 기반 유저 수, 콘텐츠 활동량 등의 시각화
- 이벤트 관리 ( 유동 ,정적 ) 수정 및 추가 삭제

## 💬 개발 목표 및 방향성

- 사용자에게 **감성적인 경험**을 제공하면서도,  
  **실용적인 기능** (추천, 일정, 알림)을 통합한 **몰입형 커플 플랫폼** 개발
- 각 컴포넌트의 UX 완성도와 **실제 사용자 입장에서의 흐름 설계**에 집중
- 추억을 ‘기록’하는 데서 그치지 않고,  
  **추천 → 일정화 → 기록화**까지 자연스럽게 이어지는 전체 흐름을 고려함

## 🧠 백엔드 기능 처리 흐름

### 🧾 1. 사용자 인증 및 권한 관리

- `Spring Security` 기반 로그인/회원가입 기능 제공  
- JWT 토큰 발급 및 `sessionStorage` 기반 사용자 인증 처리  
- `Role` 기반 권한 분리 (`USER`, `ADMIN`)  
  → 관리자만 접근 가능한 통계/삭제 기능 분기

### 🗓 2. 일정 & 날씨 알림 시스템

- 사용자가 등록한 `기념일(Anniversary)`과 `ToDo(데이트)` 일정 저장  
- `@Scheduled` 기반 스케줄러가 매일 새벽 실행되어  **3일 전** 일정 자동 감지
- 감지된 일정의 사용자의 지역 정보를 바탕으로  **OpenWeather API**를 호출해 날씨 확인
- 날씨 조건(비, 눈, 온도)에 따라 맞춤 알림 메시지 생성  
- 알림은 `WebSocket`으로 사용자에게 실시간 전달됨  
- 알림 내역은 `Alarm` 테이블에 저장되며 마이페이지에서 조회 가능

### 📍 3. 장소 추천 기능

- 사용자는 지도에서 **지역 + 카테고리(맛집, 호텔 등)** 선택
- 선택한 조건을 기반으로 `recommend_place` 테이블에서 추천 장소 조회
- 지도(`Google Maps API`) 연동 → 마커, 위치 중심 포커싱 기능 지원
- 사용자 감성을 기반으로 한 추천이 필요한 경우  

### 💬 4. AI 챗봇 연동

- `HuggingController`를 통해 Hugging Face API 호출  
- 사용자의 질문(ex. “데이트 코스 추천해줘”)을 AI 모델에 전달  
- 자연어 응답을 받아 사용자에게 리턴하는 형태  
- 간단한 대화 UX 기반 추천 경험 제공

### 📦 5. 파일 업로드 및 이미지 처리

- 장소/앨범 등에서 이미지/비디오 업로드 시, `MultipartFile`을 통해 서버에 저장
- 이미지 접근은 `/api/recommend_place/download/{파일명}` 형태로 제공
- 이미지 URL은 DB에 함께 저장되며, 프론트에서 직접 불러올 수 있음

### 📬 6. 실시간 알림 시스템 (WebSocket)

- WebSocket 기반 `/ws` 경로에서 커넥션 생성
- 사용자는 로그인 시 개인 채널로 알림 수신 대기  
- 서버 측에서 알림 발생 시 해당 사용자에게만 실시간 메시지 전송
- 댓글/좋아요/팔로우/날씨 등 다양한 이벤트에 통합 대응

---

### 🛠️ 컴포넌트 구성

프로젝트는 기능 단위로 모듈화된 컴포넌트 중심으로 구성되어 있으며,  
화면별로 역할이 분리되어 유지보수성과 재사용성을 높였습니다.

- ✅ **메인 구조**: `MainPage`, `Header`, `Footer`, `SlideMenu`
- ✅ **장소 추천 기능**: `MapPicker`, `PlaceCard`, `CategoryFilter`, `PlaceForm`
- ✅ **앨범 (기록 기능)**: `AlbumBoard`, `PhotoCard`, `AlbumDetailModal`, `AddAlbum`
- ✅ **캘린더 기능**: `AddTodo`, `Detailtodo`, `Todo`


- Zustand를 활용한 전역 상태 관리
- Styled-components로 페이지별 독립 디자인
- `sessionStorage`를 통한 인증 정보 유지 및 로그인 처리
